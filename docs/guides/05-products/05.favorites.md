# Product Favorites

> Store a recruiter's frequently used products, then fetch the full product objects when you need to render them.

## Overview

Favorites are stored per ATS user. The favorites API only manages the saved product IDs: list the IDs, add a product, or remove a product. When you need product names, pricing, logos, channel metadata, or ordering flags, call the multiple products endpoint with the `favorites` selector.

This split keeps favorite management small and lets product details continue to come from the same marketplace product response shape used elsewhere.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/v3/products/favorites/` | List favorite product IDs for the authenticated ATS user |
| POST | `/v3/products/favorites/` | Add a product to the authenticated ATS user's favorites |
| DELETE | `/v3/products/favorites/{product_id}/` | Remove a product from favorites |
| GET | `/products/multiple/favorites/` | Retrieve full product details for the authenticated ATS user's favorite products |

## Managing Favorites

List favorites:

```http
GET https://marketplace.api.vonq.com/v3/products/favorites/ HTTP/1.1
X-Auth-Token: <your Partner token here>
X-Customer-Id: customer-123
```

```json
[
  {
    "product_id": "a3e2f5d1-7c4b-4e8a-9f2d-1b3c5e7f9a0b",
    "created_on": "2026-05-12T10:15:00Z"
  }
]
```

Add a favorite:

```http
POST https://marketplace.api.vonq.com/v3/products/favorites/ HTTP/1.1
X-Auth-Token: <your Partner token here>
X-Customer-Id: customer-123
Content-Type: application/json
```

```json
{
  "product_id": "a3e2f5d1-7c4b-4e8a-9f2d-1b3c5e7f9a0b"
}
```

Remove a favorite:

```http
DELETE https://marketplace.api.vonq.com/v3/products/favorites/a3e2f5d1-7c4b-4e8a-9f2d-1b3c5e7f9a0b/ HTTP/1.1
X-Auth-Token: <your Partner token here>
X-Customer-Id: customer-123
```

Adding the same product more than once is idempotent: the API returns the existing favorite instead of creating a duplicate.

## Retrieving Product Details

The favorites API returns product IDs, not full product objects. To render favorite products in a marketplace UI, call:

```http
GET https://marketplace.api.vonq.com/products/multiple/favorites/ HTTP/1.1
X-Auth-Token: <your Partner token here>
X-Customer-Id: customer-123
```

The response is the standard paginated product list. Products are ordered by most recently favorited first.

## Search Integration

If you want favorite products pinned at the top of product search results, use `injectFavorites=true` on `GET /products/search/` with `offset=0`. This is separate from managing the favorite list:

1. Use `/v3/products/favorites/` to add or remove favorite product IDs.
2. Use `/products/multiple/favorites/` to render a favorites-only list.
3. Use `/products/search/?injectFavorites=true` to prepend favorites to the first page of a normal search result.

## Edge Cases

- Favorites are scoped to the authenticated ATS user; one user's favorites do not affect another user's list.
- A favorite is only returned if the product is still available to that ATS user.
- Favorites require ATS-user authentication. For partner-token integrations, use the secret key as `X-Auth-Token` together with `X-Customer-Id`; JWT-based ATS-user authentication is also supported where enabled.
